# New Features Implemented in Obsidian as of November 10, 2025

## Overview
As of November 10, 2025, several new features have been introduced to enhance audio handling and user interface within Obsidian, specifically through the [[Zeddal]] settings tab.

## Detailed Features

### Configurable Recordings Path
- **Settings Addition**: A new configuration option for the recordings path has been added.
- **Default Path**: A default path is now automatically set for voice notes and audio recordings.
- **User Customization**: Users can modify the location where audio files are saved via the [[Zeddal]] settings tab.

### User Interface Enhancements
- **Settings UI**: A dedicated UI section for recording settings has been implemented to improve user interaction.

### Drag and Drop Functionality
- **Implementation**: Global drag and drop capabilities have been integrated into the main TypeScript (TS) file.
- **Supported Formats**: The system now supports multiple audio formats including WebM, MP3, WAV, M4A, and OGG.
- **Automatic Saving**: Audio files dropped into Obsidian are automatically saved to the designated recordings folder.

### Recording and Transcription Improvements
- **Record Modal**: On dropping an audio file, the record modal automatically loads the existing audio, bypassing the initial recording step and moving directly to transcription options.
- **Audio Playback**: A new button for audio playback has been added, allowing users to listen to recordings directly within Obsidian.
- **Data Persistence**: Audio files and related data are now persistently saved, with added timestamps for better tracking and reference.

### Notifications
- **Toast Notifications**: Users receive real-time feedback through toast notifications during the audio processing stages.

These updates aim to streamline the audio management process and enhance the overall user experience within Obsidian.
